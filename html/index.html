Repaso de Git

Preguntas basicas

SVC o Sistema de Control de Versiones es una herramienta que nos permite registrar los cambios que atraviesa el codigo durante el desarrollo del mismo. Ademas, permite trabajar de manera colaborativa en el proyecto

Git es un SVC que se distribuye de manera gratuita y de codigo abierto. Fue creado por el mismo creador de Linux (Linus Torvalds)

Algunas alternativas a Git que son gratuitas pueden ser:
  • Gitea
  • GitBucket
  • GitPrep

Comandos de git

Comandos Uso
1 git Es el prefijo para poder trabajar con GIT
2 git init Iniciamos GIT en la carpeta donde esta el proyecto
3 git add Aniadimos los archivos para el commit. Se puede utilizar un punto (.) para aniadir TODOS los archivos
4 git commit Hace el primer commit. Podemos aniadir -m y un mensaje entre comillas que explique el commit
3 git push Sube al repositorio

Pasos a seguir para subir un proyecto a un repositorio

Nuevo repositorio Nombre Descripcion Publico o privado Subir repo Comandos
Desde nuestra cuenta de GitHub, en la parte superior derecha, dar click en el simbolo de mas (+) y elegir "New repository"
Elegir un nombre para el repositorio
(Opcional) Elegir una descripcion para el repositorio
Seleccionar si se desea que el repositorio sea publico o privado
Accede a tu terminal y desde la carpeta donde se encuentran los archivos que subiras el respositorio ejecuta los comandos siguientes:
  1. git init
  2. git add .
  3. git commit -m "comentario para el primer commit"
  4. git remote add origin https://github.com/aqui-tu-repo.git
  5. git push -u origin master
